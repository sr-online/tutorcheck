# เว็บไซต์เช็กชื่อนักเรียนติวเสริม

โครงสร้างไฟล์:

- `index.html` — หน้าเช็กชื่อสำหรับนักเรียน (มือถือ/แท็บเล็ต)
- `dashboard.html` — หน้า Dashboard สำหรับครู (เปิดดูได้เลย ไม่ต้องล็อกอิน)
- `js/data.js` — รายชื่อนักเรียนแต่ละกิจกรรม (คัดลอกจากไฟล์ PDF) แก้ไข/เพิ่มชื่อได้ที่ไฟล์นี้
- `js/firebase-init.js` — การเชื่อมต่อ Firebase
- `js/checkin.js` — ตรรกะหน้าเช็กชื่อ
- `js/dashboard.js` — ตรรกะหน้า Dashboard
- `firestore.rules` — กฎความปลอดภัยของฐานข้อมูล (นำไปวางใน Firebase Console)

## ขั้นตอนติดตั้ง Firebase (ทำครั้งเดียว)

โปรเจกต์ถูกตั้งค่าให้ใช้ Firebase project `check-in-7989c` ที่ให้มาแล้ว ไปที่
https://console.firebase.google.com/project/check-in-7989c

### 1. เปิดใช้งาน Firestore Database
Build → Firestore Database → Create database → เลือก production mode → เลือก location ที่ใกล้ (เช่น asia-southeast1)

### 2. ตั้งค่า Security Rules
Firestore Database → Rules → คัดลอกเนื้อหาจากไฟล์ `firestore.rules` ในโปรเจกต์นี้ไปวางแทนของเดิม แล้วกด Publish

(ระบบนี้ไม่ได้ใช้ Firebase Authentication เพราะข้อมูลที่เก็บ — ชื่อ/ชั้น/เวลาเช็กชื่อ — ไม่ใช่ข้อมูลอ่อนไหว จึงไม่ต้องเปิด Authentication หรือสร้างบัญชีผู้ใช้ใดๆ)

### 3. อัปโหลดเว็บไซต์
อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น hosting ที่รองรับไฟล์ static (เช่น Firebase Hosting, GitHub Pages หรือ hosting อื่นที่มีอยู่แล้ว) ไม่ต้องมีเซิร์ฟเวอร์หรือฐานข้อมูลเพิ่มเติม เพราะข้อมูลทั้งหมดเก็บบน Firestore

**ทดสอบก่อนอัปโหลด:** เปิดไฟล์ `index.html` หรือ `dashboard.html` ได้โดยดับเบิลคลิกไฟล์โดยตรง (ไม่ต้องมีเว็บเซิร์ฟเวอร์ในเครื่อง) ขอแค่เครื่องต่ออินเทอร์เน็ตได้ เพราะต้องโหลด Firebase SDK และฟอนต์จากอินเทอร์เน็ต — แต่ต้องทำขั้นตอนที่ 1-2 ให้เสร็จก่อน ไม่งั้นจะลงชื่อ/ดึงข้อมูลไม่ได้

หน้า `dashboard.html` เปิดดูได้ทันทีโดยไม่ต้องล็อกอิน — ถ้าต้องการกันไม่ให้คนทั่วไปเข้าถึงลิงก์นี้ได้ง่ายเกินไป แนะนำแค่ไม่เผยแพร่ลิงก์ตรงนี้ให้นักเรียน (เช่น ตั้งชื่อไฟล์ให้เดายาก หรือฝากลิงก์ไว้ในกลุ่มครูเท่านั้น)

## การแก้ไขรายชื่อนักเรียน / กิจกรรม
แก้ไขได้ที่ `js/data.js`
- `SUBJECTS` = รายการกิจกรรม/วิชา (แก้ชื่อ, เวลา, สี)
- `STUDENTS` = รายชื่อนักเรียนของแต่ละกิจกรรม (key ต้องตรงกับ `id` ใน SUBJECTS)

## หมายเหตุด้านความปลอดภัย
ระบบนี้เหมาะสำหรับใช้งานภายในโรงเรียน ข้อมูลที่เก็บมีเพียงชื่อ-ชั้น-เวลาที่ลงชื่อ ซึ่งไม่ใช่ข้อมูลอ่อนไหวมาก จึงตัดระบบล็อกอินออกเพื่อความง่ายในการใช้งานและติดตั้ง ทั้งหน้าเช็กชื่อและหน้า Dashboard เข้าถึงข้อมูลผ่าน Firestore Rules ที่จำกัดไว้แค่ "อ่านได้ทุกคน / สร้างข้อมูลใหม่ได้ถ้าครบฟอร์แมต / ห้ามแก้ไขหรือลบ" เท่านั้น
